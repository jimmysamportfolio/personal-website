#### OVERVIEW

# One platform to run every part of an event.

Running events at UBC BizTech means coordinating across departments, spreadsheets, Slack channels, Notion pages, and GitHub repos manually. As members of BizTech, we understood first-hand how painful disorganized and asynchronous workflows can be at scale.

What if one system understood every component of running an event, every responsibility, every action in the timeline, and then *executed* tasks on behalf of users?

## That's Nexus: the connection that brings everything together.

<Image src="/projects/nexus/landing.png" alt="Nexus landing page" width={1200} height={800} caption="Nexus landing page, nexusapp.tech" />

Nexus integrates directly into a club's existing workflow: Google Sheets, Slack, Notion, GitHub. It maps each exec's role and responsibilities, then uses a multi-agent AI system to interpret requests, break them into steps, and execute them in real time.

The primary interface is an AI chatbot engineered to understand club roles specifically. A Partnerships Director can tell the bot they just finished a meeting with a new partner, and the system will update the spreadsheet, send a Slack message, and draft an MOU as a downloadable Word doc. All autonomously.

<Image src="/projects/nexus/dashboard.png" alt="Nexus dashboard" width={1200} height={800} caption="The Nexus dashboard: mapping responsibilities to agentic workflows" />

#### HOW IT WORKS

**Frontend**: Nextjs + Tailwind with interactive animations.

**Backend**: A multi-step agentic flow that interprets tasks, compiles them, assigns them to role-specific agents (one agent per club role), and executes via API calls to Gemini, ChatGPT, and Vultr serverless inference. Connected to MongoDB for data management, with real-time integrations to Slack, Sheets, Word, Notion, and GitHub.
